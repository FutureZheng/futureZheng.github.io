算法-归并排序

2021年7月13日
| 算法

一直我都不是特别了解归并排序，或者没有太过去关注这个排序算法，导致对于它来说，感到陌生并且生疏，今天就抽空来整理一下这个排序算法的优点和特性。

以一个简单的例子来看，给定一个数组 data = [8,4,5,7,1,3,6,2]

一。思路

  1. 拆分
  2. 合并

归并排序的思路就是将一个数组，划分为多个子数组，每一个子数组，再划分子数组……..直到达到临不可划分的数组，然后将再进行数组元素的比较，将两个元素进行数组排序，排序完的数组，可以再替换原数组的所在位置顺序，直至整个数组排序完毕。

二。具体的例子

1. 拆分

将数组 data 先拆分为两个子数组（逻辑上划分）

设定数组的下标为 start = 0

数组的长度为 end = len(data) - 1

从中间开始划分数组, mid = start + (end - start ) / 2

数组可以以mid为中位线，划分 [start, mid) 和 [mid + 1, end]

然后再对子数组按照相同的方式进行划分，直到数组的下标，start 和 end 相等的情况，即数组元素个数为1,划分完毕

这个样子，一个大的数组问题，被拆分为多个细小的子数组问题，这个就是归并的第一步，拆分

2.合并

将数组逻辑上划分成为了多个子数组以后，剩下的部分，就是再这个基础上进行合并操作。

合并操作是拆分的逆过程。

合并的过程就是，给定两个数组，需要将两个子数组，排序以后，合并成为一个大的数组，下面是用局部的数组合并来分析这个问题。

下面是我用的临时数组做数组的合并操作。

给定一个数组 data, 其中 data[start:mid] 表示数组 [8]，data[mid:end]表示数组[4]

创建一个临时数组 tmp 容量等同于 data

比较两个数组的元素的大小，依次将小的元素放到临时数组tmp当中，知道某一个数组没有元素，然后将另外一个数组的元素，全部添加到tmp数组里面（因为已经排序完毕）

然后遍历tmp数组，将对应的元素，替换data对应的数组位置。

这样子，整个数组，就从拆分到合并，变成一个有序的数组。

3.时间复杂度

详细的可以参考链接地址，

https://blog.csdn.net/a1033025319/article/details/88650514

我这边的理解相对来说比较简单，比如元素8来说，它的时间复杂度多少，取决于它需要遍历的次数，在数组进行拆分的时候，不需要进行遍历，但是，在数组合并过程中，需要遍历，比如，数组8和数组4合并成数组【4，8】就需要一次遍历，数组【4，8】和数组【5，7】进行合并，又需要遍历一次元素8，数组【4，5，7，8】和数组 【1，2，3，6】合并也需要遍历一次。。。依次类推，元素8的时间复杂度为拆分的数组个数（树高），就是logn，数组元素有n个，所以归并排序的时间复杂度为O（nlogn）

4.空间复杂度

整个数组再进行排序的过程中，虽然使用到了临时数组，但是在函数调用结束以后，不会占用额外的内存空间，剩下的就是函数递归调用了，

每一次遍历，都能将数组一分为二，这样子，空间复杂度为O（logn）
