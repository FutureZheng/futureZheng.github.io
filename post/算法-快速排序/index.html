算法-快速排序

2021年5月10日
| 算法

最近再看算法中的分治思想，简单的理解就是，分而治之，比如这次学习的快排，就是将一个大的数组，分为一个基准值，和一个小于基准值的数组和一个大于基准值的数组。

然后再次对两个数组，进行相同的快速排序，直至所有的数组达到算法的临界条件，排序结束.

一。 快排的注意点

  1. 对比基准元素的选择。
  2. 数组的长度小于等于 1 的临界条件判断。

三。看看时间复杂度

看看复杂度呢？每一次排序都要遍历整个数组n,每一次遍历完，会将数组分为两个数组，平均情况下，会将元素分再不同数组里面，这样子就是O[logn]

所以快排的平均的时间复杂度O(nlogn)

但是存在一种极端的情况，就是当数据本身是有序的情况，并且是逆序的

比如：

​ 【8，7，6，5，4，3，2，1】

将其排序成 【1，2，3，4，5，6，7，8】的话，就是这种极端的情况，时间复杂度会降为O（n * n）

所以再考虑选取基准值的时候，最好先做下头部元素和中间元素的交换，避免出现这种极端情况。

四。空间复杂度

由于使用了递归调用，每进行一次快排，都将数组分为两个数组和基准值，在平均情况下，需要进行logn次递归调用，所以平均情况空间复杂度为 O(logn)

在最坏的情况下，由于没有办法把数组一份为二，所以n个元素，需要进行n-1次的递归，所以整个空间复杂度为O(n)

/**
 * 快排
 */
func quickSort(nums []int) []int {
    // 当待排序的数组小于等于1个的时候，不需要排序
    if len(nums) <= 1 {
        return nums
    }
    var (
        // index 表示基准值的起始位置
        // low表示小于基准值的位置
        // high表示大于基准值的位置
        index, low, high = 0, 0, len(nums)-1
        data = make([]int, 0, len(nums))
    )
    for low < high {
        // 找到一个小于基准值的数，表示要交换的数
        for high > low &&  nums[high] > nums[index] {
            high--
        }
        // 找到一个大于基准值的数，表示要交换的数
        for high > low  && nums[low] <= nums[index] {
            low++
        }
        if low < high{
            nums[low], nums[high] = nums[high], nums[low]
        }
    }
    // 交换基准值和最后一个小于基准值的元素，第一趟排序结束
    nums[index], nums[low] = nums[low], nums[index]

    // 根据基准值，划分小于基准值的数组和大于基准值的数组，进行局部排序
    left := quickSort(nums[:low])
    right := quickSort(nums[high+1:])

    // 将排序的结果合并成一个新的数组，排序结束
    data = append(data, left...)
    data = append(data, nums[low])
    data = append(data, right...)
    return data
}
