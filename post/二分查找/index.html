1. 二分查找算法

二分查找算法，也是一个较为常见的算法，因为算法的思想相对来说比较简单，并且效率相对来说较高。

使用的最多的就是mysql了，在innodb存储引擎当中，叶子节点存储的数据页，在数据页当中查找行数据，是通过二分查找的方式。

下面是对我自己对这个算法思想的提炼。

  • 有序数组
  • 通过数组下标，数组进行折半计算中间元素的值，将数组分为左区间，中间值，右区间
  • 比较中间元素的数值和目标数值的大小是否相等，相等则返回。
  • 根据中间值和目标值的大小来判断，目标值落于左区间还是右区间，继续第二步。直到找到元素，或者找不奥元素

这边用了变量，start 和 end，表示数组下标的起始位置和结束位置。然后计算中间元素的下标值mid。

start = 0

end = len(data) - 1 = 8

特别说明下：

​ mid 不能通过 ( start + end ) / 2 的方式来计算，因为这样子存在数值越界的风险，比如当 end 已经表示了整型的最大值，如果加上 start 后，在做除法预算，就有可能出现整型溢出的问题，所以这边采用的另外一种方式。

​ mid = start + (end - start ) / 2

通过加法的方式，来计算中间的位置，防止越界问题。

第四步

因为target < data[mid] , 所以目标值存在数组的左区间里面。

通过移动结束位置的下标来重新定位数组的区间，因为data[mid] 已经比较过了，所谓重新定位的结束下标位置

下面是具体的代码实现，采用golang预研

func binarySearch(data []int, target int) int {
	start, end := 0, len(data)
	for start <= end {
		mid := start + (end-start)/2
		if target == data[mid] {
			return mid
		} else if data[mid] > target {
			end = mid - 1
		} else {
			start = mid + 1
		}
	}
	return -1
}

特别需要再强调一次的是：二分查找适用于有序数组

/**
 * 查找算法
 */
func search(nums []int, target int) int {
   start, end := 0, len(nums)-1
   for start <= end {
      mid := start + (end-start)/2
      if target == nums[mid] {
         return mid
      }
      // 由于经过了旋转，所以没办法默认判断左右区间为有序的，所以需要优先判断是否有序
      if nums[start] <= nums[mid] {
      	 // 注意左区间为闭合区间
         if nums[start] <= target && target < nums[mid] {
            end = mid - 1
         } else {
            start = mid + 1
         }
      } else {
      	 // 注意右区间为闭合区间
         if nums[mid] < target && nums[end] >= target {
            start = mid + 1
         } else {
            end = mid - 1
         }
      }
   }
   return -1
}
